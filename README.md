# 蒲公英统一推广工作台

这是公司统一平台中的第一个通用业务板块，用于面向蒲公英全部产品与服务，持续完成推广构思、策略形成、内容策划、节奏安排、执行反馈和复盘迭代。

在线访问：<https://leariceleto.github.io/pgy-promotion-workbench/>

## 三层关系

1. 公司知识底座：公司方法论、品牌表达、业务体系、案例、历史内容和数据。
2. 推广工作台：调用知识与对象资料，完成公司级推广工作。
3. 具体长周期任务：围绕一个或多个产品、服务建立的正式推广任务。

`AI学伴`、`深度陪伴服务`、`乐高森林课程联盟`只是首轮验证对象，不是工作台的边界。

## 当前可用内容

- `app/`：可运行的推广工作台。
- `data/`：由三类原始资料生成的结构化数据。
- `00_项目定义/`：项目边界、交接文档和数据口径。
- `01_公司知识底座/`：公共知识和来源索引。
- `02_推广对象库/`：产品与服务对象的维护说明。
- `03_推广知识与历史资产/`：历史推文和推广经验沉淀说明。
- `04_长周期推广任务/`：正式任务的生命周期与确认规则。
- `05_数据与复盘/`：传播、销售和复盘口径。
- `99_历史原型/`：旧版关键项目工作台，仅作为视觉与技术参考。

## 运行方式

进入 `app/` 后运行：

```bash
npm run dev
```

生成可直接打开的单文件版本：

```bash
npm run build:standalone
```

## 当前边界

- 产品资料来自 `智库教育产品梳理7.14.docx`，当前统一标注为“原始材料”，不自动视为正式口径。
- 历史推文数据用于寻找内容线索，标题结构分类属于启发式分析。
- 商城订单只进入聚合结果，不在工作台中展示姓名、手机号、昵称和详细地址。
- AI 只接收结构化构思任务。AI 输出先进入“思考记录”，使用者明确确认后才转为正式推广任务。

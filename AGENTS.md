# 协作说明

## 项目定位

本项目必须始终遵守以下三层结构：

`公司知识底座 -> 推广工作台 -> 具体长周期推广任务`

推广工作台是公司级通用能力，不得改成某一个产品的专属工作区，也不得重新退回单纯的项目状态看板。

## 产品原则

- 首页优先支持老板完成：选对象、搭组合、定目标、想标题与内容、排推广节奏。
- 数据洞察是决策依据，不是工作台的唯一主体。
- 支持一个或多个产品、服务共同进入一轮推广构思。
- AI 能生成建议，不能直接改变正式任务状态。
- 思考记录转为正式任务前，必须由使用者明确确认。
- 事实、推断、建议需要在界面和文档中保持区分。

## 数据原则

- 所有来源保留文件名、日期、覆盖范围和处理限制。
- 产品资料未经确认前保留“原始材料”标签。
- 历史内容表现不得被表述为标题或选题的因果证明。
- 销售数据只使用聚合结果，禁止把订单个人信息放进前端或提交记录。
- 未匹配到历史推文或销售榜，不得表述为“没有推广”或“没有销售”。

## 实现原则

- 继续使用 React、Vite 和 Fluent UI，不主动更换技术栈。
- 延续安静、克制、适合管理者高频使用的企业工作台风格。
- 优先做完整工作流，避免装饰性页面和营销落地页表达。
- 本地保存使用 `localStorage`，同时保留 JSON 导出备份能力。
- 修改数据结构时，同步更新 `00_项目定义/产品结构与数据口径.md`。
